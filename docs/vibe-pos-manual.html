Section 08

Backend Crate Architecture

The Vibe POS backend is modularized into specialized Rust crates within the src-tauri/ directory, each handling a specific domain of the application logic.

graph TD Main[tauri::main] -->|Invokes| DB[database crate] Main -->|Invokes| EX[export_lib crate] Main -->|Invokes| IM[image_lib crate] Main -->|Invokes| SL[settings_lib crate] subgraph database DB --> Conn[connection.rs] DB --> Migr[migrations/] DB --> Models[Domain Models] end subgraph export_lib EX --> CSV[csv_export.rs] EX --> XLSX[xlsx_export.rs] EX --> Thai[thai_accounting.rs] end subgraph image_lib IM --> Hash[SHA-256 Hashing] IM --> Storage[Atomic Storage] end

database

The core persistence layer. Handles SQLCipher encrypted connections, Diesel ORM integration, and domain-specific logic for products, categories, recipes, and customers.

connection.rs schema.rs migrations/

export_lib

Extensible reporting engine. Supports CSV, ODS, and XLSX formats with a specialized module for Thai accounting standards and Sales Tax Reporting.

xlsx_export.rs thai_accounting.rs

image_lib

Content-addressed media management. Images are stored by their SHA-256 hash to ensure zero duplicates and atomic file system integrity.

sha2 hashing atomic writes

settings_lib

System-wide configuration manager. Handles JSON serialization, multi-version migrations, and cross-platform path resolution for application data.

migration.rs manager.rs
